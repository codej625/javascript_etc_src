# history.pushState()를 사용하여 현재 url을 변경 해보자!

1. history.pushState()란?
```
브라우저의 History API의 메서드 중 하나로, 브라우저의 기록 스택(history stack)에 새로운 상태를 추가하고 현재 URL을 변경하는 역할을 한다. 
이를 통해 페이지를 다시 로드하지 않고도 URL을 변경하고 상태를 관리할 수 있다. 
주로 웹 애플리케이션에서 라우팅 및 브라우저 내부 상태 관리를 위해 사용한다.
```

```javascript
history.pushState(state, title, url);
```

```
state: 새로운 상태를 나타내는 JavaScript 객체이다.
이 객체는 popstate 이벤트가 발생할 때 event.state를 통해 접근할 수 있다.
일반적으로 상태 데이터나 필요한 정보를 저장하기 위해 사용된다.
만약 상태 정보가 필요하지 않다면 null을 전달할 수 있다.
```
```
title: 현재는 대부분의 브라우저에서 무시되는 값이다.
보통 빈 문자열("")을 전달한다.
```
```
url: 새로운 URL을 나타내는 문자열이다.
이 URL은 현재 도메인 내에서 상대 경로("/new-url")나 절대 경로("https://example.com/new-url") 형식으로 제공할 수 있다.
```

2. 상세 설명
```
history.pushState()를 호출하면 브라우저의 주소 표시줄에 새로운 URL이 표시되며, 사용자가 뒤로 가기 또는 앞으로 가기 버튼을 클릭할 때 해당 URL로 이동하게 된다. 그러나 페이지 내용은 실제로 다시 로드되지 않는다. 
이를 통해 단일 페이지 애플리케이션(SPA)에서 URL을 관리하고, 페이지 간 전환을 부드럽게 처리할 수 있다.

다만, history.pushState()를 사용할 때에는 브라우저 내부의 상태만 변경되기 때문에, 실제 페이지 내용은 자바스크립트로 업데이트하여 표시해야 한다. 
주소 표시줄의 URL만 변경하고 싶을 때 사용되는 기술이므로, 이를 활용하여 라우팅 등을 구현할 수 있다.
```
